이 프로젝트는 UI/UX 테스트용 프론트엔드 목업이다.

- 어떤 서버도 생성하지 않는다.
- 어떤 데이터베이스도 생성하지 않는다.
- 어떤 API route도 생성하지 않는다.
- 어떤 외부 요청도 하지 않는다.
- fetch/axios 같은 네트워크 코드를 만들지 않는다.
- 환경변수를 요구하지 않는다.
- 인증 시스템을 자동 생성하지 않는다.
- 결제 SDK를 설치하지 않는다.

모든 데이터는 로컬 JSON 더미 파일에서만 가져온다.
모든 인터랙션은 로컬 상태 변화로만 시뮬레이션한다.

────────────────────────────────────────────────────────
1) 프로젝트 메타
────────────────────────────────────────────────────────
- 프로젝트명: PSF (포럼 운영 지원 모바일 앱)
- 목적: 포럼 참석자·인솔자·의전·본부 간 운영 정보를 역할별로 한 곳에서 관리하는 모바일 UI/UX 검증
- 서비스 유형: 모바일 웹앱 (프론트엔드 목업 완결 전용)
- 스택: React (Vite) + react-router-dom + recharts
- 실행: `npm run dev` → http://localhost:5173

────────────────────────────────────────────────────────
2) 로그인 방식
────────────────────────────────────────────────────────
- ID/PW 입력 폼 (역할 버튼 선택 방식 아님)
- 역할은 자격증명에 의해 자동 결정됨
- 계정은 본부에서 발급 (목업: 더미 계정 힌트 표시)
- 잘못된 자격증명: 오류 메시지 표시
- 로그인 성공 시 자동으로 /home으로 이동
- 이미 로그인된 경우 / 접속 시 /home으로 리다이렉트

더미 계정 (route_login.json):
  user1   / 1234 → 참석자, John Doe, Team A
  guide1  / 1234 → 인솔자, 김인솔, Team A
  escort1 / 1234 → 의전, 박의전, Team A
  hq1     / 1234 → 본부, 최운영, All

3) 라우터 확정 (IA 고정)
────────────────────────────────────────────────────────
1.  "/"          (public) - 로그인 (ID/PW 폼 + 더미 계정 힌트)
2.  "/home"      (auth)   - 역할별 메인 대시보드 및 퀵 액션
3.  "/notices"   (auth)   - 공지·일정 목록/상세 (본부: AI 보조 + 작성/삭제)
4.  "/attendance"(auth, 참석자 전용) - QR 출석 시뮬레이션
5.  "/gps"       (auth, 인솔자 전용) - GPS 위치 공유 더미 화면
6.  "/dashboard" (auth, 의전·본부 전용) - 출석/특이사항/인솔자 GPS 현황
7.  "/chat"      (auth, 참석자·인솔자·의전·본부) - 채팅방 리스트 및 채팅 상세
8.  "/chatbot"   (auth)   - 부산 관광 안내 AI 챗봇 더미 화면
9.  "/profile"   (auth)   - 내 프로필 확인 및 연락처/이메일 수정
10. "/contacts"  (auth)   - 역할별 필터 연락처 목록 (이름 검색 포함)
11. "/accounts"  (auth, 본부 전용) - 계정 생성/삭제 관리

- 폰트: Outfit (Google Fonts)

9) 라우터별 상세 명세
────────────────────────────────────────────────────────

[Route: "/"]
- 목적: ID/PW 로그인
- 레이아웃:
  - 로고 + 앱 설명
  - 아이디 입력 (id="login-username")
  - 비밀번호 입력 (id="login-password")
  - 로그인 버튼 (id="login-submit-btn")
  - 오류 메시지 (잘못된 자격증명)
  - 더미 계정 힌트 (클릭 시 자동 입력)
  - 우측 상단 언어 전환 드롭다운
- 클릭 시: 600ms 딜레이 → 자격증명 확인 → /home 이동 또는 오류 표시

[Route: "/home"]
- 목적: 역할별 메인 메뉴 및 퀵 액션
- 레이아웃:
  - Header (로고 + 언어 전환 + 프로필 버튼 → /profile)
  - 배너 카드 (역할 배지, 이름, 팀)
  - Quick Action 카드 (역할별 다른 액션)
    - 참석자: QR 출석, 공지 확인, 연락처
    - 인솔자: GPS 공유, 공지 확인, 연락처
    - 의전: 운영 현황, 채팅방, 연락처
    - 본부: 운영 현황, 공지 관리, 연락처, 계정 관리
  - 최근 공지 1건 미리보기
  - BottomNav

[Route: "/notices"]
- 목적: 공지·일정 확인 (본부: AI 보조 + 작성/삭제 포함)
- 레이아웃:
  - 목록 상단: 제목 + [＋ 공지 작성] 버튼 (본부만)
  - 공지 작성 폼 (본부만, 토글):
    - AI 보조 섹션 (파란 배경):
      - 주제/키워드 입력 (id="ai-draft-prompt")
      - [생성] 버튼 (id="ai-draft-btn")
      - 2초 딜레이 → 키워드 매칭 더미 응답으로 제목/내용 자동 채움
      - 키워드: 셔틀/버스/교통, 식사/밥, 일정/스케줄, 기본값
    - 제목 입력 (id="notice-title-input")
    - 내용 입력 textarea (id="notice-content-input")
    - 필독 체크박스 (id="notice-important-check")
    - 공지 등록 버튼 (id="notice-submit-btn")
  - 공지 카드 (클릭 시 내용 펼침/닫힘)
    - 필독 배지 표시
    - 삭제 버튼 (본부만, id="delete-notice-{id}")
    - 내용은 pre-line(줄바꿈 보존) 스타일

[Route: "/attendance"]
- 목적: QR 출석 체크 시뮬레이션 (참석자 전용)
- 레이아웃:
  - 카메라 스켈레톤 (어두운 박스 + 모서리 마커)
  - 내 QR 코드 더미 (7×7 격자 패턴)
  - [출석 인증하기] 버튼 (id="attend-btn")
- 상태 머신: initial → loading(1.2초) → success
- success 화면: 녹색 체크마크 + 이름 + 시간

[Route: "/gps"]
- 목적: 인솔자 위치 송신 (인솔자 전용)
- 레이아웃:
  - 지도 스켈레톤 (회색 박스, 격자선, 도로 시뮬레이션)
  - 현위치 주소 카드
  - GPS 토글 버튼 (id="gps-toggle-btn")
- 이벤트: TOGGLE_GPS
  - OFF→ON: 버튼 빨간색('송신 중지'), 지도 위 파동 애니메이션
  - ON→OFF: 버튼 파란색('위치 송신 시작'), 파동 제거

[Route: "/dashboard"]
- 목적: 출석/GPS/특이사항 현황 (의전·본부)
- 레이아웃:
  - 3칸 통계 카드 (전체/출석/미확인)
  - 도넛 차트 (recharts PieChart) + 출석률 레전드
  - 인솔자 위치 목록
    - 의전: 소속 팀 인솔자만 (라벨: "인솔자 위치 (Team X)")
    - 본부: 전체 인솔자 (라벨: "인솔자 위치 (전체)")
  - 특이사항 목록
    - 의전: 텍스트 입력창 + 추가 버튼 (id="note-input", id="note-submit-btn")
    - 본부: 조회만 (입력창 없음)

[Route: "/chat"]
- 목적: 역할별 채팅방 목록 및 메시지 뷰
- 채팅방 가시성:
  - 참석자: 소속 팀 채팅방만
  - 인솔자: 소속 팀 채팅방만
  - 의전: 의전 전체 + 준비인원 + 소속 팀
  - 본부: 준비인원만
- 레이아웃 (채팅방 상세):
  - 메시지 목록 (발신자·역할 표시)
  - 하단 입력창 + 전송 버튼

[Route: "/chatbot"]
- 목적: AI 관광/안내 응대 시뮬레이션 (전 역할)
- 퀵 질문 버튼 (관광지 추천, 다이소 위치, 식당 추천, 교통 안내)
- 이벤트: SEND_MESSAGE
  - user 말풍선 즉시 렌더링
  - 1.2초 딜레이 (쩜쩜쩜 애니메이션)
  - dummyResponses 키워드 매칭 → bot 말풍선

[Route: "/profile"]
- 목적: 내 정보 확인 및 연락처 수정 (전 역할)
- 레이아웃:
  - 상단: 아바타(역할 색상+이모지), 이름, 역할 배지, 팀
  - 계정 정보 카드 (읽기전용): 아이디, 역할, 팀
  - 연락 정보 카드 (수정 가능):
    - 연락처 (id="profile-phone")
    - 이메일 (id="profile-email")
  - [저장] 버튼 (id="profile-save-btn") → 2초 후 원래 텍스트로

[Route: "/contacts"]
- 목적: 역할별 필터된 연락처 목록 (전 역할)
- 레이아웃:
  - 이름 검색 인풋 (id="contacts-search")
  - 역할별 그룹 헤더 (본부 → 의전 → 인솔자 → 참석자 순)
  - 연락처 카드: 아바타, 이름, 팀·전화번호, 전화 아이콘 버튼
- 필터 로직:
  - 참석자·인솔자: 소속 팀 + team=All(본부)
  - 의전·본부: 전체

[Route: "/accounts"]
- 목적: 계정 생성/삭제 (본부 전용)
- 레이아웃:
  - 헤더: 제목 + [+ 계정 생성] 버튼 (id="account-create-btn")
  - 계정 생성 폼 (토글):
    - 아이디 (id="account-username")
    - 비밀번호 (id="account-password")
    - 이름 (id="account-name")
    - 역할 드롭다운 (id="account-role")
    - 팀 드롭다운 (id="account-team", 본부 선택 시 비활성화→All)
    - [계정 생성] 버튼 (id="account-submit-btn")
    - 에러: 중복 아이디 / 빈 필드 메시지
  - 계정 목록: 아이콘, 이름, 역할 배지, 아이디·팀, [삭제] 버튼
    - 삭제 클릭 시 확인/취소 버튼으로 교체 (2-step 확인)

11) QA 체크리스트
────────────────────────────────────────────────────────
□ 잘못된 ID/PW 입력 시 오류 메시지가 표시되는가?
□ 4개 역할별 로그인 후 서로 다른 홈 화면(Quick Action)이 보이는가?
