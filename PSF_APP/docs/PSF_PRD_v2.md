# PSF 포럼 앱 PRD (v2)

---

## 0. 문서 메타

| 항목 | 내용 |
|------|------|
| **프로젝트명** | PSF (포럼 운영 지원 모바일 앱) |
| **한 줄 목적** | 포럼 참석자·인솔자·의전·본부 간 운영 정보를 한 곳에서 관리한다 |
| **서비스 유형** | 모바일 웹앱 (React + Spring Boot, WebView로 Android/iOS 패키징) |
| **MVP 범위** | 역할별 로그인 → 공지·일정 확인 → 출석/GPS/특이사항 운영 → AI 챗봇 및 공지 작성 보조 |

---

## 1. 기술 환경

- **개발툴**: Antigravity (AI 코딩 어시스턴트)
- **Frontend**: React (Vite), 모바일 최적화 UI → MVP 완성 후 WebView로 Android/iOS 앱 별도 빌드
- **Backend**: Spring Boot (Spring AI 적용)
- **DB**: Supabase (PostgreSQL)
- **미디어 저장소**: Supabase Storage (QR 이미지, 첨부파일)
- **AI 연동**: Spring AI → OpenAI GPT-4o (챗봇, 공지 작성 보조)
- **지도/GPS**: 네이버 Maps API
- **결제**: 없음 (내부 운영 도구)
- **배포**: Vercel (프론트엔드) + Supabase 클라우드 (DB/Storage)

---

## 2. 핵심 차별성

- **기존 방식**: 카카오톡 단체채팅, 수기 출석부, 엑셀 특이사항 정리, 개별 전화 연락
- **PSF 차별점**:
  - 역할별 권한 분리로 정보 혼선 제거
  - QR 출석으로 수기 체크 제거
  - 인솔자 GPS 실시간 공유로 의전 대기 시간 단축
  - 팀 단위 채팅방으로 역할 간 소통 채널 구조화
  - AI 챗봇으로 부산 관광 문의를 운영진 개입 없이 처리
  - AI 공지 작성 보조로 본부의 공지 작성 시간 단축
  - 한국어·영어·프랑스어 지원으로 외국인 참석자 접근성 확보

---

## 3. 사용자 문제 정의

**현재 문제**

- 공지·일정이 카카오톡에 분산되어 참석자가 최신 정보를 찾기 어렵다
- 출석 확인이 수기로 이루어져 집계에 오류가 발생한다
- 인솔자 위치를 의전이 실시간으로 파악할 수 없어 대기 혼선이 생긴다
- 특이사항(불참, 식사 제외)이 구두·메시지로 전달되어 누락이 발생한다
- 외국인 참석자는 한국어 공지를 이해하기 어렵다
- 참석자의 관광 문의가 인솔자·의전에게 반복적으로 집중된다

**줄이려는 핵심 비용**

- 운영진의 반복 문의 응대 시간
- 출석·특이사항 집계 오류
- 인솔자-의전 간 위치 소통 비용

## 4. 사용자 흐름 (User Flow + Page Flow)

### 페이지/라우터 목록

| 경로 | 페이지 | 목적 |
|------|--------|------|
| `/` | 로그인 | ID/PW 입력 → 역할 자동 결정 (계정은 본부 발급) |
| `/home` | 홈 | 역할별 퀵 액션 메뉴 진입점 |
| `/notices` | 공지·일정 | 공지 목록 및 상세 확인 (본부: AI 보조 작성/삭제 포함) |
| `/attendance` | 출석 체크 | QR 촬영 (참석자 전용) |
| `/gps` | GPS 공유 | 인솔자 → 의전/본부 위치 공유 |
| `/dashboard` | 운영 대시보드 | 출석 현황·GPS·특이사항 (의전/본부) |
| `/chat` | 채팅 | 역할별 채팅방 목록 및 메시지 |
| `/chatbot` | AI 관광 챗봇 | 부산 관광·다이소 추천 (공통) |

| `/contacts` | 연락처 | 역할별 필터 후 연락처 목록 조회 |
| `/accounts` | 계정 관리 | 본부 전용 — 계정 생성/삭제 |

### 역할별 접근 가능 페이지

| 페이지 | 참석자 | 인솔자 | 의전 | 본부 |
|--------|--------|--------|------|------|
| 공지·일정 | ✅ 조회 | ✅ 조회 | ✅ 조회 | ✅ 작성/수정/삭제 |
| QR 출석 | ✅ | ❌ | ❌ | ❌ |
| 연락처 | 소속 팀 내 전체 | 소속 팀 내 전체 | 전체 (참석자·인솔자·의전·본부) | 전체 |
| GPS 송신 | ❌ | ✅ | ❌ | ❌ |
| GPS 수신(대시보드) | ❌ | ❌ | ✅ 소속 팀 인솔자만 | ✅ 전체 인솔자 |
| 대시보드 | ❌ | ❌ | ✅ | ✅ |
| 특이사항 기재 | ❌ | ❌ | ✅ | ✅ 조회만 |
| 의전 전체 채팅방 | ❌ | ❌ | ✅ | ❌ |
| 준비인원 채팅방(의전+본부) | ❌ | ❌ | ✅ | ✅ |
| 팀 채팅방 | ✅ 소속 팀만 | ✅ 소속 팀만 | ✅ 소속 팀만 | ❌ |
| AI 챗봇 | ✅ | ✅ | ✅ | ✅ |

### 핵심 흐름

1. **참석자**: 로그인 → 홈 → 공지 확인 / QR 출석 / 팀 채팅방 / AI 챗봇
2. **인솔자**: 로그인 → 홈 → 공지 확인 / GPS 송신 / 팀 채팅방 / AI 챗봇
3. **의전**: 로그인 → 홈 → 대시보드(출석·팀내 GPS) / 특이사항 기재 / 의전 전체 채팅방 + 준비인원 채팅방 + 팀 채팅방 / AI 챗봇
4. **본부**: 로그인 → 홈 → 공지 작성(AI 보조) / 대시보드(전체 GPS) / 준비인원 채팅방

> **계정 관리**: 본부가 계정을 직접 생성·발급·삭제하며, 역할 및 팀 배정도 본부가 담당한다.

## 6. MVP 성공 기준

- [x] 4개 역할(참석자·인솔자·의전·본부)으로 로그인 및 권한 분리가 정상 동작한다
- [x] 본부가 계정을 생성하고 역할·팀을 지정할 수 있다 (목업: /accounts 페이지)
- [ ] 참석자가 QR을 촬영하면 출석이 DB에 기록되고 대시보드에 반영된다 ※ 미구현(목업: 로지만 시뮬레이션)
- [ ] 인솔자의 GPS 위치가 네이버 지도 기반으로 의전·본부 대시보드에 표시된다 ※ 미구현(목업: 스켈레톤+주소 텍스트)
- [x] 본부가 공지 작성 시 AI 초안 생성 버튼을 누르면 2초 이내 초안이 반환된다 (목업: 키워드 매칭 더미 응답)
- [x] AI 챗봇에 부산 관광지 또는 다이소를 묻는 질문 입력 시 관련 답변이 반환된다
- [x] UI 주요 텍스트가 한국어·영어·프랑스어로 전환된다
- [x] 의전이 특이사항을 기재하면 구경 대시보드에 즉시 표시된다 (목업: 로컬 상태)
- [x] 의전 전체 채팅방 및 팀별 채팅방에서 메시지 송수신이 정상 동작한다
- [x] 참석자·인솔자는 소속 팀 연락처만 조회되고, 의전은 본부 포함 전체 연락처가 조회된다 (목업: /contacts 페이지)

## 8. 구현 Gap 리포트 (목업 기준, 2026-03-10)

> 목업 프론트엔드가 PRD와 다르게 구현되었거나 미구현된 항목을 기록한다.
> 배경: 백엔드(Spring Boot + Supabase) 연동 전 UI/UX 검증 단계이므로, 네트워크 요청이 없는 로컬 시뮬레이션으로 대체한다.
